1

User-Interaktion

• The own sphere can be moved and rotated through space by mouse and keyboard
o “W” – Forward
o “S” – Backward
o “A” – Left
o “D” – Right
o “Mouse Y” – X Axis
o “Mouse X” – Y Axis
• Exit:
o “Cancel” – "Esc"

2

Objektinteraktion

Spheres collide with each other. The bigger sphere absorbs the smaller one. The Volume that gets transfered is based on how big and close ths spheres are.
RandomSpheres collide with the border of the playing ground and bounce of it.
The controllableSphere cannot cross the Border either.
Based on the distance and rotation between the controllableSphere and the other randomSpheres the alpha channel of the mass text for every random sphere is set, so that the further away the other spheres are, the more transparent the taxt becomes.
The same counts for the rotation, so that the text is always the right way around, even when the controllableSphere is upside down.
Based on how big the controllableSphere is, the randomSpheres change color from red to green and back.

3

Objektanzahl variabel

Spheres will be generated in the beginning of the game according to given parameters.
The amount, size and speed can be adjusted

4

Szenenhierarchie

The Camera is inside of the controllableSphere, to be in a 3rd person view.
The controllableSphere is inside of the gameManager, to make use of the messaging system.
All used Canvas are stacked inside of each other, to ensure a scalable view of the HUD

5

Sound

• Every time a randomSphere interacts with the controllableSphere a sound plays. The sound stays as long as the spheres are touching.
• The touching-sound fades in and out.
• The touching-sound changes pitch, depending on the size of the two interacting volumes
• Music is played

6

Timeline Animation

In the main menu, an animation of an interchanging cube and sphere is shown. It demonstrates a changing material, too.
The sizes of the spheres are directly animated in the code.

7

GUI

• Before the game begins, a user Interface exists to start and explain the Game.
• During the game the interface shows the current current Volume of the controllableSphere.
• The interface also displays a slider that mass the mass of the controllableSphere compared to the whole existing mass.
• A mass text shows the mass of every randomSphere on every sphere

8

Externe Daten

From the json file, that will be loaded: • maxPlayingField --> Size of the Cube where the spheres are in.
• snapValue --> Size under which a sphere ceases to exist.
• massTransferDivisor --> Divisor of the amount of mass transfered per second. The bigger, the less mass gets transfered.
• maxTransSpeed --> Speed of the controllableSphere.
• maxRotSpeed --> Rotation speed of the controllableSphere.
• amountOfSpheres --> Amount of random spheres, that spawn in the beginning.
• maxRadius --> The randomSpheres will have a random Radius between 0 and maxRadius.
• minSpeed --> The RandomSpheres will have a random speed between minSpeed and maxSpeed.
• maxSpeed --> The RandomSpheres will have a random speed between minSpeed and maxSpeed.

9

Statische Klasse

SettingsLoader: Singelton pattern, loads the settings from the json file during the menu. The settings are then available ingame. Declaration: "public void getSetting(string variableName, ref float toSet)"
gameManager.sphereList: Contains a reference to all spheres on the scene.
gameManager.mySphere: Contains a reference to the controllableSphere.
sphereRules.wholeVolume: has the amount of mass of all spheres.

All of the above settings values are static, because they are the same for every instance.
An InformationKeeper gameObject makes use of the "DontDestroyOnLoad" methode. It passes the endMessage to the menu. But after passing the message it destroys itself.

10

Serverkommunikation

Retriving the Json file over an URL

11

Maße & Positionen

The size of objects is, what makes this game. Object size is higly dynamic and relative to each other.
The Position of objects is relative to the playingField.
The only non relative point of this game is the playingField, which starts as (0,0,0) and expands to maxPlayingField in all three axis.

12

Message System

The message System has been used to cross reference some childs and parents with each other. (broadcastMessage, sendMessageUpwards)
Also it has been used to sent an instantiated sphere its initial Radius(sendMessage). More importantly, the simmilar system "getComponent<>()" has widely been used to get, for example, the <sphereRules> object out of a collision. In that way you are guaranteed to have certain methodes, that you can call on a GameObject. Further examples are: Get the image of a slider foreground, to set the color. Get the renderer of a sphere to change the color

13

Lichteinsatz

In Menu: Simple directional light for the buttons
In the Game: Point light in the middle of the playing field to create a space feeling
